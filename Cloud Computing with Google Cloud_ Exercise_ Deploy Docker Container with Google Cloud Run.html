    Cloud Computing with Google Cloud

    Done: View

    4. Exercise: Deploy Docker Container with Google Cloud Run

    What You'll Do

    • In this exercise, you will continue to use the Google Cloud Shell.
    • You'll learn how to use it to deploy a basic Docker container to Google Cloud Run.


    Instructions 

    Set up hello-google container

    Configure and upload the Django app to Cloud Shell 

    • Download the code for a basic Django app below:

    hello-google.zip

    • Do not extract its contents yet.
    • Like the other "hello world"-type apps you have used so far, this is a minimal Django app that, when running, responds to HTTP GET requests to a specific port with a webpage. 
    • The zipfile comes with a provided Dockerfile for building a Docker image for this Django app.
    • We will next upload this zipfile to the virtual machine accessed through Cloud Shell. 
    • In your Cloud Shell, click the three-dots menu at the upper right of the Shell, and choose Upload:


    • In the Upload modal that pops up, leave File selected and click Choose Files.

    • After clicking Choose Files, you will be able to select a file from your filesystem. Find the hello-google.zip file that you downloaded and select it for upload.
    • You should now see the filename appear under the Choose Files button, as shown below. 
    • Leave everything else at its default settings. The Destination Directory should show /home/<your username>:

    • Click the UPLOAD button.
    • Back in the Cloud Shell bash prompt, enter this command to unzip the uploaded file:
    unzip -q hello-google.zip
    • This should have extracted the Django app along with its corresponding Dockerfile to a folder named hello-google.
    • Enter the commands:
    cd hello-google
    ls

    • You should have been able to enter the hello-google/ folder and list its contents, as follows:

    • If you do not see a result similar to the above, then something has gone wrong with either your upload or unzip. Retrace your steps and see if you can troubleshoot what went wrong. 
    • Otherwise, continue on.
    • Realize that at this point, we have the code for a basic Django app and a Dockerfile for creating a Docker image, uploaded to the Cloud Shell virtual machine. 
    • We are close to where we need to be. We have these tasks ahead of us:
      • Build a Docker image from the Django app and Dockerfile 
      • Store the image in a Google Container Registry
      • Using that image, deploy a container using Google Cloud Run
    • We can use the gcloud utility in Cloud Shell to accomplish all these tasks.

    Confirm environment setup

    • In your Google Shell, make sure of the following things before you continue:
    • First, make sure that your current folder is the hello-google/ folder, and that when you enter ls, the output looks like this:

      • If you don't see this, then move to the hello-google folder with the command:
    cd ~/hello-google
      • Then enter ls again and see if you can now see the same files as in the screenshot above.
    • Second, enter the following two commands:
    echo $PROJECT_ID
    echo $REGION
      • ...and make sure that the result shows your own Project ID and the region you set earlier in this exercise. Example:

    • If you had to restart your session since you initially set these values, the values will have been erased.
    • If you do not see any values set for this variables, set them again:

    PROJECT_ID=$(gcloud config get-value core/project)
    REGION=us-central1
    • Afterward, you can confirm that the values were set correctly by using the two echo commands once again.
    • Next, we'll build a Docker image in the Google Container Registry, using Google Cloud Build.


    Build Docker image and deploy container with Google Cloud Run

    • Enter this gcloud command in your Google Shell:
    gcloud builds submit --tag gcr.io/$PROJECT_ID/hello-google:v1

    • You may see a request to Authorize Cloud Shell to make a Google Cloud API call on your behalf. Go ahead and authorize it.


    • If the build is working, you will see an initial result similar to this:

    • When the image is finished being built, you should see a confirmation message similar to this:

    • This step is not necessary, but if you like, you can see for yourself in the Google Container Registry that your image exists there by visiting: https://console.cloud.google.com/gcr
    • Finally, we will deploy the image to Cloud Run with the following command. All three lines of the command below should be copied and pasted into the Cloud Shell terminal together:
    gcloud run deploy hello-google --port 8000 --region $REGION \
  --image gcr.io/$PROJECT_ID/hello-google:v1 \
  --allow-unauthenticated

    • If all goes well, you should see a message similar to this after that moment:
